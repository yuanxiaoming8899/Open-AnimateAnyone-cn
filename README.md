<p dir="auto"><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">虽然我们无法使用大规模数据来训练模型，但我们可以提供几个训练建议：</font></font></p>
<ol dir="auto">
<li><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">在我们的实验中，AnimateAnyone原论文中的poseguider无论我们使用什么激活函数（如ReLU、SiLU）都很难控制pose，但输出通道放大到320并添加在conv_in之后（如model .hack_poseguider ）非常有效，同时，与 controlnet 相比，该解决方案更加轻量级（&lt;1M para vs 400M para）。</font><font style="vertical-align: inherit;">但我们仍然认为Controlnet是一个不错的选择。</font><font style="vertical-align: inherit;">Poseguider 依赖同时进行微调的unet，无法立即使用。</font><font style="vertical-align: inherit;">即插即用。</font></font></li>
<li><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">在小规模数据集中（少于 2000 个视频），stage1 可以很好地工作（包括泛化），但 stage2 需要大量数据。</font><font style="vertical-align: inherit;">当数据量较低时，很容易出现伪影和闪烁。</font><font style="vertical-align: inherit;">因为我们在第一阶段重新训练了unet，原来animatediff的checkpoint失去了作用，所以这一阶段需要大量高质量的数据集来重新训练animatediff的运动模块。</font></font></li>
<li><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">冻结unet并不是一个好的选择，因为它会丢失参考图像的纹理信息。</font></font></li>
<li><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">这是一项需要大量数据的任务。</font><font style="vertical-align: inherit;">我们相信，扩大数据质量和规模通常比修改模型的微小结构更有价值。</font><font style="vertical-align: inherit;">数据的数量和质量非常重要！</font></font></li>
<li><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">高分辨率训练非常重要，它影响细节的学习和重建。</font><font style="vertical-align: inherit;">训练分辨率不应大于推理分辨率。</font></font></li>
</ol>
